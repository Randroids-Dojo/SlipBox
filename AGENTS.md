# SlipBox Agent Instructions

## Writing style

- **Never use em-dashes (`—`, U+2014)** anywhere: code comments, JSDoc, string
  literals, UI copy, docs, commit messages, or generated note content. This is
  a hard rule. Use a colon for a lead-in (`Edge colors: relation types`), a
  comma or parentheses for an aside, or a spaced hyphen (` - `) where a break
  is genuinely needed. En-dashes (`–`) are allowed only in numeric ranges
  (e.g. `0–1`).
- Prefer plain ASCII punctuation in code and docs.
